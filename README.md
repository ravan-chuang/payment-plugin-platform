# Dynamic Payment Plugin Platform

## Project Overview
This project demonstrates a plugin-based checkout architecture in C for integrating payment methods and discount strategies without modifying or recompiling the host application.

Payment and discount implementations are compiled as independent shared libraries. The host application reads plugin configurations, dynamically loads selected plugins at runtime, validates required symbols, applies discounts, processes payments, and writes transaction logs.  
  
## Key Features

- Runtime loading of payment and discount shared-library plugins
- Separate interfaces for payment methods and discount strategies
- Configuration-based plugin discovery
- Required-symbol validation before execution
- Credit card, LINE Pay, PayPal, and bank-transfer plugins
- No-discount, student-discount, and festival-discount plugins
- End-to-end checkout flow with subtotal, discount, and final payment
- Successful and rejected transaction handling
- Transaction logging
- Makefile-based build workflow
- Automated checkout, runtime-extension, and validation demos
- Broken-plugin test for validating defensive error handling

## Technologies
- C, Linux, Shared library, dlopen, dlsym, Makefile, Bash

## Usage

List enabled payment methods:

    ./shop_app list

Show payment plugin metadata:

    ./shop_app info

Validate all registered plugins:

    ./shop_app validate

Process a direct payment:

    ./shop_app pay credit_card 1200
    ./shop_app pay line_pay 800
    ./shop_app pay paypal 2000
    ./shop_app pay bank_transfer 1500

Run checkout with discount and payment plugins:

    ./shop_app checkout student_discount credit_card
    ./shop_app checkout festival_discount line_pay
    ./shop_app checkout no_discount paypal
    ./shop_app checkout festival_discount bank_transfer

Run full e-commerce demo:

    ./demo.sh

Run runtime plugin extension demo:

    ./runtime_extension_demo.sh

Run plugin validator demo:

    ./validator_demo.sh

## Configuration

Payment plugins are configured in plugins.conf:

    credit_card=enabled=./build/plugins/credit_card.so
    line_pay=enabled=./build/plugins/line_pay.so
    paypal=disabled=./build/plugins/paypal.so

Discount plugins are configured in discounts.conf:

    no_discount=enabled=./build/discounts/no_discount.so
    student_discount=enabled=./build/discounts/student_discount.so
    festival_discount=enabled=./build/discounts/festival_discount.so

## Checkout Flow

    Cart Items
    -> Subtotal
    -> Load Discount Plugin
    -> Apply Discount
    -> Final Amount
    -> Load Payment Plugin
    -> Process Payment
    -> Write Transaction Log

## Core Dynamic Linking Functions

- dlopen(): loads a shared library at runtime.
- dlsym(): finds required symbols inside the loaded plugin.
- dlclose(): closes the loaded shared library.

## Limitations

- This project simulates payment behavior and does not connect to real payment APIs.
- Cart items are fixed for demonstration.
- Transaction IDs are generated using Unix timestamp.
- Plugin signature verification and sandboxing are not implemented.
- The project focuses on Linux dynamic linking, runtime extensibility, and plugin-interface design rather than production payment processing.
